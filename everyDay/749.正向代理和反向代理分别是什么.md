# Problem: 正向代理和反向代理分别是什么?

*[interview]: start
正向代理和反向代理是两种不同的代理服务器架构，它们的作用和工作方式有所不同。

1. **正向代理（Forward Proxy）**：
   - 正向代理是位于客户端和原始服务器之间的代理服务器，它代表客户端向原始服务器发起请求。在这种情况下，客户端知道目标服务器的存在，但服务器不知道请求的源自客户端，只知道请求来自代理服务器。
   - 正向代理的主要作用是隐藏客户端的真实 IP 地址，保护客户端的隐私，并且可以绕过某些网络限制和审查。举例来说，翻墙工具就是正向代理的一个典型应用。

2. **反向代理（Reverse Proxy）**：
   - 反向代理是位于原始服务器和客户端之间的代理服务器，它代表服务器向客户端返回响应。客户端知道代理服务器的存在，但不知道后面的原始服务器。对于客户端来说，所有的请求和响应都是与代理服务器直接交互的。
   - 反向代理的主要作用是将客户端的请求转发到多个后端服务器（负载均衡），以提高性能和可用性；同时可以隐藏后端服务器的信息，增加系统的安全性。另外，反向代理还可以提供缓存、SSL 终端、安全过滤等功能。

总的来说，正向代理是代表客户端发起请求，而反向代理是代表服务器处理请求。**正向代理隐藏了客户端的真实身份，而反向代理隐藏了服务器的真实身份**。两者在功能和作用上有着不同的应用场景。

## webpack中proxy 和vite中的Proxy是正向代理还是反向代理
Webpack中的proxy和Vite中的Proxy都属于正向代理。

在Webpack中，通过配置devServer中的proxy选项，可以实现在开发环境下对API请求的代理转发。这意味着当开发环境中的前端应用向API发送请求时，这些请求会先经过Webpack配置的proxy代理服务器，然后由代理服务器再将请求转发给后端API服务器，最终返回结果给前端应用。

同样地，在Vite中的Proxy配置项也是用于实现前端应用发送请求时经过代理服务器的功能，这里的Proxy也是正向代理，将前端应用发送的请求代理到其他服务器上，然后将结果返回给前端应用。

总的来说，这两个工具中的Proxy都是用作正向代理，将前端应用的请求代理到其他服务器上。
